    class PSSRAS(builtins.object)
     |  PSSRAS(name, condition=None, action=None, enabled=True, report=None)
     |  
     |  Represents a Remedial Action Scheme (RAS)
     |  
     |  Methods defined here:
     |  
     |  __init__(self, name, condition=None, action=None, enabled=True, report=None)
     |      Creates a new RAS definition.
     |      name: Name of RAS (must be unique)
     |      condition: Condition function to be called when checking if this RAS
     |      is triggered. This function may take arguments and must return a
     |      boolena value (True or False)
     |      action: Action function to be called when implementing this RAS
     |      when triggered. This function may take arguments and may return
     |      anything or None (returned value is not used).
     |      enabled: True (armed) or False (disarmed)
     |      report: (optional) custom report function for RAS reports and logs. If
     |      provided, must be a function that takes no arguments and returns a
     |      string.
     |      return: New RAS object
     |  
     |  check_condition(self, *args, **kwargs)
     |      Check the RAS condition and sets the triggered flag if the condition
     |      is met.
     |  
     |  get_report(self)
     |      Gets the user-defined custom report string for this RAS.
     |      return: Custom report string if exists, otherwise None.
     |  
     |  implement_action(self, *args, **kwargs)
     |      Implements the RAS function.
     |  
     |  reset(self)
     |      Resets the state of this RAS (sets triggered and executed flags to
     |      False).
     |  
     |  ----------------------------------------------------------------------
     |  Data descriptors defined here:
     |  
     |  __dict__
     |      dictionary for instance variables (if defined)
     |  
     |  __weakref__
     |      list of weak references to the object (if defined)

    define_ras(name, condition, action, enabled=True, report=None, description=None)
        Defines a RAS.
        name: The user-defined name of the scheme.
        condition: The name of a defined function that will be called each
        ras iteration to determine if this ras is triggered. This function should
        return True for triggered or False otherwise.
        action: The name of the function that will be called if this ras is
        triggered.
        enabled: (optional) True for enabled (armed) or False for disabled
        (disarmed).
        report: (optional) a function that returns a string for custom
        reporting.
        description: (optional) detailed description of the RAS.
        returns: The Scheme object created.

    post_ca_soln_adj(conlabel, solved)
        This is an entry point to the RAS action routine. This function
        should be called by the user-defined pssuserpf.py module's post_ca_soln_adj
        function like this:
        
        [in pssusrpf.py]
        import pssras
        ...
        def post_ca_soln_adj(conlabel):
            pssras.perform_actions(conlabel)
    
    post_ca_soln_eval(conlabel, solved)
        This is the entry point to the RAS check routine. This function
        should be called by the user-defined pssuserpf.py module's
        post_ca_soln_eval function like this:
        
        [in pssusrpf.py]
        import pssras
        ...
        def post_ca_soln_eval(conlabel):
            solved = pssras.condition_check(conlabel)
            return solved
        
        It is important that the pssuserpf function gets and returns the "solved"
        flag. This is the "all clear" flag that is set to true if no RAS conditions
        are found which require adjustments and a re-solve.
